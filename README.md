# task-cli

Personal user story and task tracker — a Go CLI that uses LLMs to generate structured user stories, tasks, and subtasks stored locally in SQLite.

No cloud sync. No accounts. Your data stays in `~/.task/`.

---

## Install

```bash
make install
```

This builds the binary, installs it to `~/.local/bin/task`, and installs the man page to `/usr/local/share/man/man1/task-cli.1`.

Or build manually:

```bash
go build -o task ./cmd/task
```

After installing, the full manual is available via:

```bash
man task-cli
```

### Homebrew

```bash
brew tap pooch1e/task-cli https://github.com/pooch1e/homebrew-task-cli
brew install task-cli
```

## Data model

```
Project
 └── Story  (S-1, S-2 …)        "As a user, I want to …"
      └── Task   (T-1, T-2 …)   concrete implementation steps
           └── Subtask (T-1.1)  optional fine-grained checklist
```

Slugs are project-scoped and never reused after deletion. Stories hold acceptance criteria as Given/When/Then scenarios generated by the LLM.

## Quick start

No API key needed — use the free `pi` provider via GitHub Copilot:

```bash
# 1. Initialise in your repo
cd ~/your-project
task init

# 2. Generate a story
task story "add OAuth login" --agent pi --model github-copilot/claude-haiku-4.5

# 3. See what was created
task list

# 4. Start working
task start T-1

# 5. Mark done
task done T-1
task status
```

## Configuration

Config lives at `~/.task/config.toml` (chmod 600).

```bash
task config init    # create with defaults
task config show    # view current settings
task config test    # verify your LLM connection
```

Example config:

```toml
[llm]
provider     = "deepseek"
model        = "deepseek-chat"
base_url     = "https://api.deepseek.com/v1"
# api_key    = "sk-..."         # or set TASK_API_KEY env var
max_tokens   = 1024
timeout_secs = 30

[storage]
db_path = "~/.task/tasks.db"

[obsidian]
vault_path = ""                 # set with: task obsidian set-vault <path>
```

### LLM providers

| Provider | Cost / 1K tokens | Key required | Notes |
|---|---|---|---|
| `pi` | free | none | GitHub Copilot subprocess |
| `opencode` | free | none | GitHub Copilot subprocess |
| `deepseek` | ~$0.00011 | `TASK_API_KEY` | Default provider |
| `openai` | varies | `OPENAI_API_KEY` | GPT-4o etc. |
| `gemini` | varies | `GEMINI_API_KEY` | Google Gemini |

Run `task providers` to see the full list with current provider highlighted.

### Environment variables

| Variable | Purpose |
|---|---|
| `TASK_API_KEY` | LLM API key (all paid providers) |
| `TASK_PROVIDER` | Override provider |
| `TASK_MODEL` | Override model |
| `TASK_BASE_URL` | Override API base URL |

## Workflows

### 1. Start a new project

```bash
cd ~/my-project
task init
task story "user can sign up with email"
task story "user can reset password"
task list
```

### 2. Track daily work

```bash
task list                   # see everything
task start T-3              # begin a task
task done T-3               # finish it
task done S-1               # mark the whole story done
task status                 # progress summary
```

### 3. Review a story in detail

```bash
task show S-2               # title, description, tasks, acceptance criteria
```

### 4. Add work manually (no LLM call)

```bash
task add task "Write load tests" --story S-1
task add subtask "Test 1000 concurrent users" --task T-5
```

### 5. Remove a story or task

```bash
task rm S-3                 # prompts for confirmation
task rm T-7 --yes           # skip confirmation
```

### 6. Export to a file

```bash
task export                              # markdown to stdout
task export -f json                      # JSON to stdout
task export -f markdown -o backlog.md    # write to file
```

### 7. Export to Obsidian

Connect your stories to an Obsidian vault as interlinked Markdown files with wikilinks and backlinks visible in Graph View.

```bash
# One-time setup
task obsidian set-vault ~/Documents/MyVault

# Export (run again whenever you want to sync new stories)
task obsidian export
```

Vault layout:

```
MyVault/
  my-project/
    S-1 Add OAuth login/
      story.md            ← description, AC, links to tasks
      tasks/
        T-1 Implement OAuth flow.md
        T-2 Write login tests.md
    S-2 User can sign up/
      story.md
      tasks/
        T-3 Build signup form.md
```

Each `story.md` links to its tasks. Each task file links back to its story and lists subtasks inline.

Existing files are **never overwritten** — edits you make in Obsidian are preserved. To regenerate a file from the database, use `--force`:

```bash
task obsidian export --force
```

> Note: Obsidian export is one-way. Edits made in Obsidian do not sync back to the database.

## Command reference

| Command | Description |
|---|---|
| `task init` | Register the current repo as a project |
| `task story "<desc>"` | Generate story + tasks via LLM |
| `task list [-p <project>]` | List all stories and tasks |
| `task show <S-N>` | Show story with acceptance criteria |
| `task status` | Progress summary (counts + percentages) |
| `task start <T-N>` | Mark task as in-progress |
| `task done <T-N\|S-N>` | Mark task or story as done |
| `task add task "<title>" --story <S-N>` | Add a task manually |
| `task add subtask "<title>" --task <T-N>` | Add a subtask manually |
| `task rm <S-N\|T-N> [--yes]` | Remove story or task (and children) |
| `task export [-f markdown\|json] [-o file]` | Export to file or stdout |
| `task providers` | List LLM providers |
| `task config init` | Create default config file |
| `task config show` | Print config (key redacted) |
| `task config test` | Test LLM connection |
| `task obsidian set-vault <path>` | Save Obsidian vault path to config |
| `task obsidian export [--vault <path>] [--force]` | Export to Obsidian vault |
| `task version` | Print version |

For per-command detail:

```bash
task help <command>
man task-cli
```

## Data & privacy

- All data is stored in `~/.task/tasks.db` (SQLite, chmod 600)
- Config at `~/.task/config.toml` (chmod 600)
- No telemetry, no network calls except to the configured LLM provider
- Open with any SQLite viewer: TablePlus, DB Browser, `sqlite3`
